# Internationalization

Tool names and descriptions can be customized or translated to better suit your workflow or language preferences.

## Overview

The GitLab MCP Server supports customizing tool descriptions through a JSON configuration file. This allows you to:
- Translate tool descriptions to your preferred language
- Customize descriptions for your team's workflow
- Make descriptions more specific to your use case

## How It Works

1. **Generate Translation Template**: Run the server with `--export-translations` flag
2. **Customize Descriptions**: Edit the generated `gitlab-mcp-server-config.json` file
3. **Restart Server**: The server automatically loads translations on startup

## Generating Translation Template

Create a configuration file with all available translation keys:

```bash
./gitlab-mcp-server stdio --export-translations
```

This creates `gitlab-mcp-server-config.json` in the same directory as the binary with all 60+ translation keys.

## Configuration File Format

The configuration file uses a simple key-value format:

```json
{
  "TOOL_GET_PROJECT_DESCRIPTION": "Retrieves details of a specific GitLab project.",
  "TOOL_LIST_PROJECTS_DESCRIPTION": "Returns a list of GitLab projects with optional filtering.",
  "TOOL_GET_ISSUE_DESCRIPTION": "Retrieves details for a specific GitLab issue.",
  "TOOL_CREATE_ISSUE_DESCRIPTION": "Creates a new issue in a GitLab project."
}
```

## Configuration File Location

The server looks for `gitlab-mcp-server-config.json` in:

1. **Standalone binary**: Same directory as the binary
2. **Docker**: Mount the config file to `/app/gitlab-mcp-server-config.json`

### Docker Example

```bash
docker run -i --rm \
  -v $(pwd)/gitlab-mcp-server-config.json:/app/gitlab-mcp-server-config.json \
  -e GITLAB_TOKEN=<your-token> \
  gitlab-mcp-server:latest
```

## Loading Translations

Translations are loaded automatically when the server starts:

1. Server checks for `gitlab-mcp-server-config.json` in the binary directory
2. If found, translations are loaded and applied
3. If not found, default English descriptions are used
4. Invalid JSON or missing keys fall back to defaults

## Best Practices

### Translation Guidelines

1. **Keep descriptions concise**: Tool descriptions should be clear and brief
2. **Maintain technical accuracy**: Ensure translations accurately describe tool functionality
3. **Use consistent terminology**: Use the same terms across related tools
4. **Test translations**: Verify translations work correctly after changes

### Team Workflows

1. **Share configuration files**: Commit `gitlab-mcp-server-config.json` to your repository
2. **Version control**: Track changes to translations
3. **Documentation**: Document any custom terminology or workflow-specific descriptions

## Troubleshooting

### Translations Not Loading

**Problem:** Custom descriptions not appearing

**Solution:**
- Verify file is named `gitlab-mcp-server-config.json`
- Check file is in the correct location (same directory as binary)
- Verify JSON syntax is valid
- Check file permissions
- Restart the server after making changes

### Invalid JSON

**Problem:** Server fails to start or translations don't load

**Solution:**
- Validate JSON syntax using a JSON validator
- Check for trailing commas
- Ensure all strings are properly quoted
- Verify file encoding is UTF-8

### Missing Keys

**Problem:** Some tools still show default descriptions

**Solution:**
- Regenerate the template with `--export-translations`
- Ensure all keys are present in the config file
- Check key names match exactly (case-sensitive)
